### Реализовать меню игры

Задачи:
- разработать дизайн меню игры, выводимого на экран с помощью текста;
- спроектировать подпрограмму, печатающую меню на экран и ведующую диалго с пользователем.

Основные функции подпрограммы:
1) выводить на экран макет меню;
2) выводить на экран приглашение выбрать один из вариантов:
    - начать новую игру;
    - выйти из игры.
3) считывать и обрабатывать ввод пользователя с клавиатуры. Обрабатывать варианты:
    - начать новую игру;
    - выйти из игры.
    - неверный ввод;
4) в зависимости от выбора пользователя:
    - запускать подпрограмму нового раунда;
    - посылать основной программе сигнал к завершению.


### Реализовать операции внутри отдельного раунда

Задача: спроектировать подпрограмму, имитирующую один теннисный матч по заданным правилам.

Основные функции подпрограммы:
1) отслеживание и вывод на экран номера текущего раунда;
2) выводить на экран приглашение выбрать один из вариантов:
    - в раунде побеждает игрок 1;
    - в раунде побеждает игрок 2;
    - завершить игру;
3) считывать и обрабатывать ввод пользователя с клавиатуры. Обрабатывать варианты:
    - выбор игрока 1;
    - вабор игрока 2;
    - завершить игру;
    - неверный ввод;
4) в зависимости от выбора игрока:
    - обновлять счет игроков -- печатать результаты раунда -- переходить к анализу соотношения очков
    - выходить в главное меню игры
    - выполнять повторное приглашение ввода с клавиатуры


### Реализовать хранение и обновление очков игроков

Задача: подобрать и спроектировать подходящую структуру данных для хранения очков каждого игрока.

Структура данных должна позволять:
1) хранение значений;
2) доступ к хранящимся значениям;
3) перезапись значений одной подпрограммой и сохранение обновленных данных для считывания другой подпрограммой.


### Реализовать обход дерева сценариев

Задача: разработать подпрограмму для анализа текущего соотношения очков у игроков.

Основные функции подпрограммы:
1) считывание очков каждого игрока;
2) обход дерева сценариев и определение реализованного варианта;
3) в зависимости от реализованного сценария возвращать определенный параметр, определяющий дальнейшие процедуры внутри основной программы.

### Реализация завершения раунда в зависимости от реализованного сценария

Задача: разработать подпрограмму инициализации дальнейших процедур в зависимости от сценария внутри раунда.

Основные функции подпрограммы:
1) "победа по правилам тенниса" -- инициализация выхода в меню игры;
2) "переход к следующему раунду по правилам тенниса" -- вызов подпрограммы нового раунда.
